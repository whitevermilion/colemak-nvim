LazyVim 的 `plugins` 文件夹是框架的核心组件库，包含预配置好的插件模块。以下是主要插件分类和代表性插件（基于最新 LazyVim 结构）：

### 核心插件目录结构
```
plugins/
├── alpha.lua             # 启动页
├── autopairs.lua         # 自动括号
├── bufferline.lua        # 标签栏
├── cmp.lua               # 代码补全
├── colorscheme.lua       # 主题系统
├── comment.lua           # 注释工具
├── dap.lua               # 调试器
├── editor.lua            # 编辑增强
├── flash.lua             # 快速跳转
├── git.lua               # Git集成
├── illuminate.lua        # 高亮相同词
├── indent.lua            # 缩进线
├── keymaps.lua           # 键位映射
├── lsp.lua               # LSP核心
├── lualine.lua           # 状态栏
├── mason.lua             # LSP管理器
├── neo-tree.lua          # 文件树
├── noice.lua             # 通知系统
├── notify.lua            # 通知美化
├── nvim-tree.lua         # 备用文件树
├── project.lua           # 项目管理
├── scrollbar.lua         # 滚动条
├── smart-splits.lua      # 窗口管理
├── surround.lua          # 包围操作
├── telescope.lua         # 模糊搜索
├── toggleterm.lua        # 终端管理
├── treesitter.lua        # 语法高亮
├── trouble.lua           # 诊断面板
├── which-key.lua         # 快捷键提示
└── ...                   # 其他
```

### 插件加载机制

```mermaid
graph LR
    A[init.lua] --> B[加载 plugins/*.lua]
    B --> C[按需初始化插件]
    C --> D[延迟加载非核心插件]
    D --> E[用户配置覆盖]
```

### 特点优势

1. **开箱即用**：所有插件预配置最佳实践
   ```lua
   -- telescope.lua 示例
   require('telescope').setup{
     defaults = {
       sorting_strategy = 'ascending',
       layout_config = {
         prompt_position = 'top'
       }
     }
   }
   ```

2. **模块化设计**：按需禁用/替换
   ```lua
   -- 用户配置中禁用
   { "folke/flash.nvim", enabled = false }
   ```

3. **性能优化**：智能加载策略
   ```lua
   event = "BufReadPost"  -- 文件打开后加载
   cmd = "Telescope"      -- 命令触发加载
   ```

4. **依赖管理**：自动处理插件关系
   ```lua
   dependencies = {
     "nvim-lua/plenary.nvim"  -- telescope 依赖
   }
   ```

### 用户定制
在 `lua/config/lazy.lua` 中添加：
```lua
return {
  { import = "plugins" },  -- 加载所有核心插件
  { "user/custom-plugin" } -- 添加个人插件
}
```

这种结构使 LazyVim 在保持 40ms 内启动的同时，提供 IDE 级功能体验。所有插件都经过精心配置和测试，确保协同工作无冲突。
